# 版本控管與 Git/GitHub 使用指南

這份文件說明 Git (版本控制工具) 與 GitHub (雲端託管平台) 的基本概念、常用指令以及 `.gitignore` 的運作方式。

## 1. Git 與 GitHub 的關係與運用

簡單來說：
*   **Git** 是在您的電腦（**本機**）上運作的工具，負責記錄檔案的每一次修改。
*   **GitHub** 是網路上的平台（**遠端**），用來儲存您用 Git 記錄下來的檔案與歷史紀錄。

### 運作流程
1.  **本機開發**: 您在電腦上寫程式，並使用 Git 記錄修改 (Commit)。
2.  **上傳備份**: 將這些記錄 **推送 (Push)** 到 GitHub 上。
3.  **同步更新**: 如果在不同電腦工作，可以從 GitHub **拉取 (Pull)** 最新進度下來。

## 2. 常用指令

打開終端機 (Terminal) 或 VS Code 的 Terminal 介面使用。

### 初始設定 (專案剛開始時)

```bash
# 1. 初始化: 在目前資料夾啟用 Git
git init

# 2. 連結遠端: 設定 GitHub 的倉庫網址 (origin 是遠端倉庫的代稱)
# 請將 URL 替換成您實際的 GitHub Repo 網址
git remote add origin https://github.com/AbauDog/novel-listen.git

# (選用) 檢查目前的遠端連結
git remote -v

git push -u origin main

git pull origin main --allow-unrelated-histories


### 日常開發循環 (最常用的三個指令)

每次完成一個階段的工作，通常會依序執行這三步：

```bash
# 步驟 1: 加入 (Stage)
# 將目前的變更檔案加入準備提交的名單
git add . 
git status 
# 步驟 2: 提交 (Commit)
# 將步驟 1 加入的檔案存成一個版本，並附上說明
git commit -m "聽小說線上版"


# 查看狀態: 隨時可看目前哪些檔案被修改、哪些還沒加入
git status
# 查log
git log --since="2025-12-21" 

# 強制還原
git reset --hard 2d4bc824972024e089f74e641d1f0a78dee410c1   #將所有已追蹤的檔案強制回復到該版本，捨棄所有修改。
git clean -fd   #強制刪除所有未追蹤的新增檔案與資料夾

# 下載更新: 從 GitHub 下載最新的檔案到本機
# (通常在換電腦工作，或是多人合作時使用)
git pull origin main

# 複製專案: 從 GitHub 下載整個專案 (包含所有歷史紀錄)
git clone https://github.com/AbauDog/novel-listen.git
```

## 3.  .gitignore

*   `.gitignore` 是一個純文字檔，裡面列出了「Git 應該忽略的檔案清單」。例如編譯產生的 `build/` 資料夾、個人的設定檔、或是敏感的金鑰檔案。
*   1.  當您在本機 Commit 並 Push 到 GitHub 時，`.gitignore` 也會被上傳。
*   2.  當您（或其他人）從 GitHub 下載 (Clone/Pull) 專案時，也會拿到這份 `.gitignore`。
*   3.  這確保了所有參與這個專案的人（以及所有的電腦），都遵循同一套「忽略規則」，不會不小心把暫存檔或垃圾檔案上傳上去。
